# Examples

Three end-to-end runs of the nl2protocol pipeline, ordered by increasing complexity. Each subdirectory contains the input the system was given, the Opentrons Python script the system generated, the simulator output confirming the script runs, and the pipeline's debug log.

The point of this directory is to show what the system actually produces without requiring you to install and run it.

## What's here

| # | Directory | Demonstrates |
|---|-----------|--------------|
| 1 | [`01_simple_transfer/`](./01_simple_transfer/) | Direct extraction of an instruction with explicit volumes, wells, and tip strategy. Three transfers compressed to one logical step, expanded back into three `transfer()` calls. |
| 2 | [`02_pcr_mastermix/`](./02_pcr_mastermix/) | Mid-complexity protocol with multi-pipette routing, distribute-vs-transfer operation selection, and per-sample mixing. |
| 3 | [`03_bacterial_transformation/`](./03_bacterial_transformation/) | Complex 12-step protocol with temperature-module orchestration, heat-shock cycle, multiple pauses, dual pipettes, and per-reagent tip discipline. |

## File layout in each example

- `instruction.txt` — the natural-language input.
- `lab_config.json` — the lab config the LLM was given (deck layout, labware, pipettes, modules).
- `generated_protocol.py` — the Opentrons script the system produced.
- `simulation_log.txt` — Opentrons simulator output confirming the script runs.
- `pipeline_state.json` — the dev-only debug log; per-stage state including provenance traces, gap detection, refinement attempts, and validation results. Useful for understanding what the system did internally.
- `run_output.txt` — the CLI's stage-by-stage stdout (examples 1 and 3 only; example 2 was reused from a prior successful run that didn't capture stdout — `pipeline_state.json` carries the equivalent information in structured form).

## Reproducing these locally

After installing nl2protocol (see the project [README](../README.md)), each example can be regenerated by running:

```bash
cd examples/03_bacterial_transformation
nl2protocol -i instruction.txt -c lab_config.json
```

Note that LLM extraction is non-deterministic — the exact generated script may differ between runs even with identical inputs. The scripts checked in here are representative outputs from successful runs; the system's verification stages (provenance check, hardware constraints, simulation, intent verification) are what guarantee any generated script is sound.
